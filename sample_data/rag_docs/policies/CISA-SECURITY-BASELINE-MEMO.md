# Chief Information Officer Directive — Baseline Protections *(synthetic sample)*

**CDO-2024-07**  
**To:** Component CIOs and Authorizing Officials  
**Subject:** Minimum security controls for cloud collaboration tools

## Background

Agencies increasingly rely on SaaS collaboration platforms for mission delivery. This directive aligns internal configuration baselines with NIST SP 800-53 moderate-impact controls for systems handling operational data.

## Required controls (summary)

1. **Identity.** Phishing-resistant MFA for all privileged roles; step-up authentication for contractor accounts.
2. **Data protection.** Default **deny** external sharing; DLP rules for SSN and financial account patterns.
3. **Logging.** Centralized audit logs retained 90 days online and 1 year in cold storage; correlation IDs on API calls.
4. **Vulnerability management.** Monthly automated scans; critical findings remediated within 15 days or documented risk acceptance.

## Exceptions

Exceptions require AO sign-off, compensating controls, and entry in the agency Plan of Action and Milestones (POA&M) register.
